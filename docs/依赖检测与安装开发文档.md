# 依赖检测与安装开发方案

| 任务 | 任务内容 | 测试方法 |
| --- | --- | --- |
| 1. 明确依赖准入规则 | 应用启动时按顺序检查 Node.js、Git；Node.js 不满足时不再检查 Git。仅当两个工具均可执行且版本满足要求时，应用才进入主界面。Node.js 的最低兼容版本固定为 18（即 Node.js 18+）；Git 至少须可执行并能返回版本。Claude Code 不属于启动门禁，改在首次进入项目页时单独检查。依赖不满足时，窗口的最小化、最大化、关闭仍可用，但 Tab、快捷键和主界面功能必须不可用。 | 在开发机分别执行 Node.js、Git 的版本命令，确认“可执行且版本合格”“可执行但版本不合格”“不存在或无法执行”三类结果都有明确判定。 |
| 2. 实现后端依赖检测 | 在 Rust 后端新增统一的依赖检测能力，返回结构化结果：依赖名称、状态（已安装、缺失、版本不兼容、检测错误）、路径、版本和面向用户的错误说明。检测不能只依据 PATH 中是否存在同名文件，必须实际启动版本命令。检测错误必须与“未安装”区分，避免权限、损坏的可执行文件或异常 PATH 被误报为未安装。 | 使用正常的 Node.js/Git 环境验证已安装结果；在隔离环境或测试替身中验证缺失、不可执行、旧版本和命令执行失败。不得通过修改生产检测代码、重命名真实 `node.exe` 等方式模拟场景。 |
| 3. 实现启动门禁与生命周期衔接 | 在 `App.vue` 中增加单一依赖状态：检测中、Node.js 缺失、Git 缺失、版本不兼容、检测错误、安装中、需要重启、已就绪。首次渲染显示“正在检测”，不能短暂误显示“未安装”。窗口状态恢复、主题、关闭清理和事件监听等基础生命周期必须始终注册；配置、项目、终端等业务组件仅在“已就绪”后挂载，避免依赖不满足时提前初始化。 | 观察正常启动时没有错误提示闪现；Node.js 缺失时只显示 Node.js 门禁且不检查 Git；Node.js 通过而 Git 缺失时切换到 Git 门禁；关闭、最小化、最大化、窗口状态恢复、主题恢复仍正常；两个依赖满足后验证配置、项目、终端的既有功能。 |
| 4. 实现下载入口和一键安装 | 对 Node.js、Git 缺失页提供固定的官网链接和 winget 安装按钮。官网链接使用现有 shell 插件打开固定 URL，不增加可传任意地址的后端命令。执行 winget 前先检测其可用性，并向用户明确展示软件包、将接受的软件包/源协议以及可能的管理员授权。安装命令在后台运行，界面显示真实的“安装中”“成功”“失败”状态；禁止重复发起同一安装，也不能假设 winget 会弹出独立窗口或立即返回。 | 在有 winget 的开发环境验证两条安装命令的参数和成功/失败反馈；在无 winget、网络不可用、用户取消安装、权限被拒绝、包安装失败等场景中验证友好错误提示和官网入口仍可使用。 |
| 5. 实现安装完成后的重启流程 | Node.js 或 Git 安装成功后，状态必须进入“需要重启应用”，主界面仍保持锁定，并明确提示用户关闭后重新打开应用。手动从官网下载并安装完成后也采用同一流程。当前进程不得承诺可以凭“重新检测”立即获得新 PATH；重新启动后才重新执行完整检测和主界面初始化。 | 在干净 Windows 环境中依次安装 Node.js、Git：每次安装完成但未重启前，确认应用不会错误放行；重新启动后确认检测到新版本。两个依赖完成后进入主界面，并实际创建终端/启动既有流程，验证新环境可被后续子进程继承。 |
| 6. 完成端到端验收与回归 | 先完成静态检查，再使用打包后的 Windows EXE 在干净虚拟机或测试机验收。缺 Node.js 的场景不能使用 `npm run tauri dev` 测试，因为该命令本身依赖 Node.js。验收覆盖：两个依赖均已安装、仅缺 Node.js、仅缺 Git、两者均缺失、Node.js 版本不兼容、winget 不可用、安装失败、安装后重启、手动安装后重启，以及窗口和既有主界面回归。 | 开发机执行 TypeScript 和 Rust 静态检查；使用打包 EXE 在干净 Windows 虚拟机完成场景表并记录结果。验收通过标准是：不满足依赖时无法使用主界面，错误信息可操作，安装后只有重启才放行，且依赖满足后原有配置、项目、终端、窗口状态功能均无回归。 |
| 7. 首次进入项目页检查 Claude Code | 首次请求进入项目页时，后端定位 Claude Code 候选路径并实际执行版本检测。若可用，正常进入项目页，后续切换不重复检测；若缺失或无法执行，显示覆盖项目内容的全屏提示页，提供官方安装说明、一键执行 `npm install -g @anthropic-ai/claude-code`、重新检测和返回配置。一键安装完成后按钮切换为“重新检测”。配置页中原有的 Claude Code 状态检查复用同一后端能力。 | 已安装 Claude Code 时首次进入项目页正常显示，后续切换不重复检查；未安装时项目内容无法操作且显示提示页；一键安装完成后右侧按钮变为“重新检测”，点击后可进入项目页；手动安装后同样可重新检测；从配置页内置终端启动 Claude Code 的原有流程保持可用。 |
